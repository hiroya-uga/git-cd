# Implementation Reference

This document compares the current Bash and PowerShell implementations of `install` and `git-cd`.

For user-facing behavior and installation details, see [README.md](../README.md) (common), [BASH.md](../BASH.md) (Bash / Zsh), and [POWERSHELL.md](../POWERSHELL.md) (Windows). This document focuses on implementation differences and platform-specific decisions that inform test and code changes.

Use it when you need to:

- confirm whether behavior is intentionally shared across platforms
- check where the implementations differ
- update tests or docs after changing one side

## Files Covered

- `install / Bash`: `install.sh`
- `install / PowerShell`: `install.ps1`
- `git-cd / Bash`: `bin/git-cd`
- `git-cd / PowerShell`: `bin/git-cd.ps1`
- `Windows Launcher`: `bin/git-cd.cmd`

Notes:

- The Bash-side implementation files are `install.sh` and `bin/git-cd`, and both are Bash scripts.
- On Windows, `bin/git-cd.cmd` helps launch the PowerShell script, but persistent directory changes are achieved by the `function git` added to the PowerShell profile.
- Entries marked as `Same` in the tables mean the prerequisite or behavior is effectively identical.
- Observations are stated in terms of the current implementation files and their visible behavior.

## install

### Preconditions

| Item                                                           | Bash                                                                                                                                                               | PowerShell                                                                                                              |
| -------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------- |
| Implementation language / execution assumptions                | `install.sh` is a Bash script and assumes execution under `bash`, using values such as `BASH_SOURCE`. By default it updates either `~/.bashrc` or `${ZDOTDIR:-$HOME}/.zshrc`, and `-zl` / `--zshrc-local` override the target to `${ZDOTDIR:-$HOME}/.zshrc.local`. | `install.ps1` is a PowerShell script and updates the PowerShell profile at `$PROFILE`.                                  |
| Files required in a cloned checkout                            | Assumes `bin/git-cd` exists.                                                                                                                                       | Assumes `bin/git-cd.ps1` and `bin/git-cd.cmd` exist.                                                                    |
| Download source when adjacent implementation files are missing | If `bin/git-cd` cannot be found next to `install.sh`, it downloads `git-cd` from GitHub Releases.                                                                  | If `bin/git-cd.ps1` and `bin/git-cd.cmd` cannot be found next to `install.ps1`, it downloads both from GitHub Releases. |
| Install destination                                            | `~/.local/bin`                                                                                                                                                     | `$env:USERPROFILE\bin`                                                                                                  |
| Shell config file to update                                    | If `-zl` or `--zshrc-local` is passed, `${ZDOTDIR:-$HOME}/.zshrc.local`; otherwise if `basename "${SHELL:-}"` is `zsh`, `${ZDOTDIR:-$HOME}/.zshrc`; otherwise `~/.bashrc` | `$PROFILE`                                                                                                              |
| Optional dependency                                            | `fzf` is optional. Installation still succeeds without it.                                                                                                         | Same                                                                                                                    |

### Behavior

| Item                                    | Bash                                                                                                                                                                                                     | PowerShell                                                                                                                                                                                                                                          |
| --------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Install method from a clone             | Creates a symbolic link at `~/.local/bin/git-cd`. Future updates from `git pull` are reflected automatically.                                                                                            | Copies `git-cd.ps1` and `git-cd.cmd` to `$env:USERPROFILE\bin`. Later repository updates are not reflected in the installed copies unless `install.ps1` is run again.                                                                               |
| Install method for standalone execution | Downloads `git-cd` with `curl -fsSL` and makes it executable with `chmod +x`.                                                                                                                            | Downloads `git-cd.ps1` and `git-cd.cmd` with `Invoke-WebRequest`.                                                                                                                                                                                   |
| PATH update                             | Appends `export PATH="$HOME/.local/bin:$PATH"` only if the rc file does not already contain the string `.local/bin`.                                                                                     | Appends `$env:USERPROFILE\bin` to the user `PATH` environment variable only if it is not already present.                                                                                                                                           |
| Hook for `git cd`                       | Appends a `git()` shell function. Only when the subcommand is `cd`, it runs `command git-cd "$@"` and then `builtin cd` to the returned path. All other subcommands are delegated to `command git "$@"`. | Appends a PowerShell `function git`. Only when the subcommand is `cd`, it runs `git-cd` and then `Set-Location` to the returned path. All other subcommands are delegated to the executable resolved by `Get-Command git -CommandType Application`. |
| Re-running with existing config         | Replaces the block between `# git-cd BEGIN` and `# git-cd END`, avoiding duplicate additions.                                                                                                            | Same                                                                                                                                                                                                                                                |
| Install timestamp                       | Writes `# Installed: YYYY-MM-DD HH:MM:SS` into the rc file.                                                                                                                                              | Writes `# Installed: YYYY-MM-DD HH:MM:SS` into the profile.                                                                                                                                                                                         |
| When the config file does not exist yet | Even if `~/.bashrc`, `.zshrc`, or `.zshrc.local` does not exist, it is created by appending to it.                                                                                                      | Creates the parent directory and file for `$PROFILE` if necessary before appending.                                                                                                                                                                 |
| Completion message                      | Tells the user to open a new terminal tab and suggests installing `fzf` with Homebrew on macOS or with the system package manager on Linux.                                                              | Tells the user to open a new terminal and suggests `winget install fzf`.                                                                                                                                                                            |
| Idempotency                             | Can be re-run without duplicating the same configuration block.                                                                                                                                          | Same                                                                                                                                                                                                                                                |

## git-cd

### Preconditions

| Item                           | Bash                                                                                                                                                          | PowerShell                                                                                                                                                                                                                |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Default search root            | Uses `git config --global --get git-cd.root` if available; otherwise uses `$HOME`.                                                                            | Uses `git config --global --get git-cd.root` if available; otherwise uses `$env:USERPROFILE`.                                                                                                                             |
| Required environment variables | Assumes `HOME` when determining the cache path and default root. There is no explicit check for it being unset.                                               | Requires `USERPROFILE`; if unset, it exits with `USERPROFILE environment variable is not set`. In addition, `LOCALAPPDATA` is implicitly required when building the cache path, and execution fails there if it is unset. |
| Invocation style               | When run as a standalone script, it prints the selected path to standard output. Persistent directory changes happen via `git cd` through the shell function. | When run as a standalone script, it prints the selected path to standard output. Persistent directory changes happen via `git cd` through the PowerShell profile function.                                                |
| Optional dependency            | `fzf` is optional. If it is not installed, a fallback numbered-selection UI is used.                                                                          | Same                                                                                                                                                                                                                      |

### Behavior

| Item                                  | Bash                                                                                                                                                                                                                                                                            | PowerShell                                                                                                                                                                             |
| ------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Basic behavior                        | Searches for git repositories and writes the selected path to standard output. The calling shell function then runs `cd` to that path.                                                                                                                                          | Searches for git repositories and writes the selected path to standard output. The calling PowerShell function then runs `Set-Location` to that path.                                  |
| Path argument                         | A positional argument can override the search start location. If multiple positional arguments are passed, the last one interpreted as such takes effect. The typical usage is `git cd [path]`.                                                                                 | Same                                                                                                                                                                                   |
| Help                                  | `-h`, `--help` prints usage and exits. Usage is written to standard error.                                                                                                                                                                                                      | `-h`, `--help` prints usage and exits. Usage is written to standard output.                                                                                                            |
| Depth limit                           | `--depth <n>` limits traversal depth. The default is `5`.                                                                                                                                                                                                                       | Same                                                                                                                                                                                   |
| Nested gitfile repo discovery         | Repositories whose `.git` is a file are excluded by default and included only when `--nested` is specified. Regular nested repositories with a `.git` directory are still discovered normally.                                                                                  | Same                                                                                                                                                                                   |
| Cache usage                           | With `--cache`, if a cache already exists it is reused without re-scanning. Without it, scanning is performed every time and the cache is overwritten.                                                                                                                          | Same                                                                                                                                                                                   |
| Cache path                            | A single file at `~/.cache/git-cd`                                                                                                                                                                                                                                              | `%LOCALAPPDATA%\git-cd\cache`                                                                                                                                                          |
| Special handling of the `cd` argument | If the resolved search start location is the literal string `cd` and there is no real directory with that name, it is treated as the subcommand name passed through a git alias and falls back to the default root. If such a directory exists, it is treated as a normal path. | Same                                                                                                                                                                                   |
| Non-existent path                     | Prints `Not a directory: <path>` and exits with status code `1`.                                                                                                                                                                                                                | Same                                                                                                                                                                                   |
| No repositories found                 | Prints `No git repositories found under <path>` and exits with status code `1`.                                                                                                                                                                                                 | Same                                                                                                                                                                                   |
| With `fzf`                            | Sends the candidate list to `fzf` and returns the selected result.                                                                                                                                                                                                              | Same                                                                                                                                                                                   |
| Without `fzf`                         | Prints the candidate list to standard error via `cat -n`, reads a number from standard input, and returns the corresponding path.                                                                                                                                               | Prints the candidate list in `1) path` format, reads a number with `Read-Host`, and returns the corresponding path.                                                                    |
| Excluded directories                  | Excludes `node_modules`, `Library`, and `.Trash` from traversal.                                                                                                                                                                                                                | Excludes `node_modules`, `.git`, `AppData`, `$RECYCLE.BIN`, and `System Volume Information` from traversal. `.git` is included to prevent recursing into git internals.                |
| Nested repo detection                 | If `.git` is a directory, it is treated as a regular repository; if it is a file, it is treated as a nested gitfile-based repository such as a submodule. File-type `.git` entries are excluded unless `--nested` is specified.                                                 | Same                                                                                                                                                                                   |
| Search implementation                 | Uses `find` to search for `.git`.                                                                                                                                                                                                                                               | Uses a recursive PowerShell function to detect the presence of `.git`.                                                                                                                 |
| Windows launcher                      | Not applicable.                                                                                                                                                                                                                                                                 | `git-cd.cmd` prefers `pwsh.exe` if available, otherwise launches `git-cd.ps1` with `powershell.exe`. `cmd.exe` alone cannot preserve a current-directory change in its parent process. |
